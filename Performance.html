1.1 INTRODUCTION

As a class, we are developing an application designed to help faculty better understand equity in student outcomes.

Our performance group is responsible for creating an interactive tab that allows users to explore DFW rates (D,F, and Withdraw) and GPA across selected student groups.

This tab allows faculty to filter and compare outcomes by variables such a gender, race, department, course level and the class standing.

For this component, we used student IDs as unique IDs, converted letter grades into numeric GPA values and created variables such as DFW rate and course level classifications. We created a format with which we can filter lower and upper divison courses. All newly created variables were tested and no missing values and AU values were found.

1.2 Data

The registrar dataset was imported and several cleaning and transformations were done.

First, we created a DFW indicator variable. Grades A through C and Pass were coded as 1, and grades of D,F AND W were coded as 0.

Then, we converted the letter grades into numeric GPA values using 4.0 grading scale. This made it easier to compute mean GPAs and summary across different groups. Grades marked as AU were recorded as missing vales so they would not affect the GPA or DFW calculations.

We then created two structural variables related to course level:

• Upper/Lower Division Course:

Courses numbered in the 100-200 range were labeled as lower division courses and the courses in the 300-600 range were labeled as upper division.

• Course level (crs_level) :

We created categories such as 1XX,2XX,3XX, etc. This column contains no missing values and is used directly for the filtering in the app.

Additional cleaning steps were included in removing commas from class standings and non-alphabetic charaters were removed from race/ethnicity fields. The final dataset was saved as cleaned_data.csv which is the main dataset for all the analysis and visualizations.
